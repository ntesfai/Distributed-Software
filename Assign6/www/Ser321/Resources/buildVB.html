Configure Virtual Box and Debian Stretch (9.x) Linux

The goal is to configure Debian Stretch Linux operating system running under Virtual Box on your laptop or desktop computer. This platform will be used for program development for all of the exercises and examples of the course. All examples and assignment solutions have been created, compiled and executed on this platform.

  • Download and Install Virtual Box. Obtain the appropriate installation file from Virtual Box Downloads. You will also want to download the appropriate extension pack which can also be found on the download page. Install Virtual Box using the downloaded installation file and following the directions for you host type found at: https://www.virtualbox.org/manual/ch02.html.

  • Download the Linux Debian Installation CD Image. Download the first CD image of Debian 9.x (Stretch). See https://www.debian.org/releases/stretch/ for release information. The instructor's machine uses: debian-9.3.0-amd64-xfce-CD-1.iso from the page http://cdimage.debian.org/debian-cd/9.3.0/amd64/iso-cd/. For the current Stretch release you may need to start at: http://cdimage.debian.org/debian-cd/. If your are running on Windows, you may need to make changes to enable 64-bit virtualization. See:
    • Virtual Box Forum: I have a 64-bit host, but can't install 64-bit guests.
    • HP: How to Enable Intel Virtualization Technology.

  • Create the Virtual Machine. Startup Virtual Box and under Machine menu, select New. You will have to complete several options. Here are some reasonable choices for completing the work in this class:
    • 1024 MB memory
    • Create a dynamically allocated Virtual Disk Image (VDI) with about 10 GB space.
    • Point to the iso image: debian-9.3.0-amd64-xfce-CD-1.iso
    • Use your ASURiteID as the user name
    • Partition: guided using the entire disk
    • Partitioning Scheme: All files in 1 partition
    • Write To Disk: Yes
    • Network Mirror: Yes. Select one in the Western US.
    • Grub boot loader: Yes
    • Software to install: use the default, or another desktop environment if you wish. We'll be installing other software as part of the course.

  • Reboot or restart with the installation CD removed.

  • Give your user administrative (sudo) privilege. After logging in with your asuriteid, start up an xterm terminal, and as super-user (su) execute the adduser commands below and then restart.
    su -
    apt-get install sudo
    adduser asuriteid sudo
    exit

  • Install Guest Additions. To better integrate the guest Debian system with your host operating system, install VB Guest Additions. Add guest additions by loading the Guest Addtions CD and following directions found at: https://www.virtualbox.org/manual/ch04.html. First you will need to add additional sofware to your Debian installation. Before doing so, update the system with:
    sudo apt-get update
    sudo apt-get upgrade
    On my installs, I had to also install the following packages before Guest Additions:
    sudo apt-get install dkms build-essential linux-headers-$(uname -r)
     Get VB to insert and mount the Guest Additions CD. Devices-->Insert-Guest-Add... After the CD is mounted an icon appears on the desktop. (To remove a mounted CD, click the CD icon at the bottom of VBox window.) Now open up a terminal and change to the Guest Addtions CD with:
    cd /media/cdrom0
    ls -l
    sudo sh ./VBoxLinuxAdditions.run
    You will need to wait for the installation to complete. Its best to restart the machine after installation with:
    sudo shutdown -r now
    There are directions at: https://www.turnkeylinux.org/docs/virtualbox-guest-addons.

  • Install software. Login as your user, and install desired software packages using apt-get, such as:
    sudo apt-get install ant g++ libjsoncpp-dev libjsonrpccpp-dev
    This will install Ant, C++, and Json support for C++. To use apt-get, see: apt-get quick help. At some point you should also install the following packages (if not already installed): libfltk1.3-dev, git, ant-doc, ant-contrib, ant-contrib-cpptasks, emacs, vim, gedit, doxygen, and lxterminal. Note that most of these packages also need to be installed on your Raspberry Pi in the same manner. On the RPi, however, you do not need to install doxygen (documentation generator for C/C++), lxterminal, or libfltk1.3-dev (full-light toolkit of UI components for C++).

  • Install java9.
    • On the Raspberry Pi this can be installed with:
      sudo apt-get install openjdk-9-jdk
    • To install it on VBox, use one of the following:
      • Use the Oracle packages made available through the WebUpd8 Oracle PPA repository. To make them visible to your VBox machine perform the commands below (which are taken from Install Oracle 9). From a terminal execute:
        su -
        echo "deb http://ppa.launchpad.net/webupd8team/java/ubuntu xenial main" | tee /etc/apt/sources.list.d/webupd8team-java.list
        echo "deb-src http://ppa.launchpad.net/webupd8team/java/ubuntu xenial main" | tee -a /etc/apt/sources.list.d/webupd8team-java.list
        apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys EEA14886
        apt-get update
        exit
        Searching packages now will find the oracle java packages, for example:
        apt-cache search oracle | grep -i java9
         Now install Oracle jdk9 with
        sudo apt-get install oracle-java9-installer

      • Or, Install Oracle jdk9 manually by agreeing to the lincense and downloading the Linux tar.gz archive from the Oracle Java SE Development Kit 9 Downloads page. Save the archive into your home directory. Extract it with:
        tar zxvf jdk-9.0.4_linux-x64_bin.tar.gz
        Note that the file name above was current at the time this page was created and will change with future releases of jdk9. This will create the directory jdk-9.0.4 in your home directory. To provide bash with access to the jdk9 tools execute (again from your home directory -- be sure to substitute your userid for myasuriteid in the line below):
        cat >> .bashrc
        export PATH="/home/myasuriteid/jdk-9.0.4/bin:$PATH"
        ^C

  • Install optional software. If you don't want to learn emacs, or vi, then you may want to install gedit, which is a window-based text editor that runs on Linux. Install with:
    sudo apt-get install gedit
    One example we'll be reading and running uses all of the following languages: C, C++, Java, and C#. You won't be required to develop in either C or C#. You can get a full C# compiler and runtime with:
    sudo apt-get install mono-complete

  • Optionally Create a Share. Provide access to your user files on your host system. In Virtual Box, go to
    Devices-->Shared Folders Settings-->Shared Folders
    Add a new share to your home directory on the host file system. On the instructor's system, the share is mounted in VB Linux with the shell command:
    sudo mount -t vboxsf -o rw,uid=1000,gid=1000 macHome ~/MacHome/
